## Iterator

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


/**
 * 什么是迭代？
 * 
 * 
 * 从一个数据集合中按照一定的顺序，不断取出数据的过程
 */


//-------------------------------------------------------------------------------------------------------------------//


/**
 * 迭代和遍历的区别
 * 
 */

迭代: 迭代器强调的是依次取数据，并不保证取出多少，也不保证把所有数据取完

遍历: 遍历强调的是把整个数据依次全部取出


//-------------------------------------------------------------------------------------------------------------------//


/**
 * 迭代器( Iterator )
 * 
 * 
 * 对于迭代过程的封装，在不同语言中有不同表现形式的迭代器( 通常为对象 )
 */


//-------------------------------------------------------------------------------------------------------------------//


/**
 * 迭代模式
 * 
 * 
 * 一种设计模式，用于统一迭代过程，并规范了迭代规则
 */

1) 迭代器应具有获取下一个数据的能力

2) 迭代器应具有判断是否还有后续数据的能力


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```

#### Iterator of JavaScript

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


/**
 * Iterator
 * 
 * JavaScript 规定，若一个对象具有 next 方法，并且该方法返回一个对象，该对象的格式如下
 */

const obj = {

    next() {                // 获取下一个数据的方法
    
        return {
        
            value: nextValue,           // 下一个数据值
            done: boolean               // 判断是否迭代完成
        }
    }
}


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```
